API-dokumentasjon

Interaktiv demo av API er tilgjengelig p ekstern.dev.nav.no. Hvis du kjrer en sprring i den interaktive demoen, s sendes et ekte nettverkskall. Du kan derfor bruke developer-tools for se hvordan JSON-et til foresprselen og svaret ser ut.

Autentisering

Tjenesten deres m autentisere seg med Azure AD, type serverserver, som beskrevet i nais-dokumentasjonen.

Endepunkter for miljer

milj url
mock https://notifikasjon-fake-produsent-api.ekstern.dev.nav.no/api/graphql

prod https://ag-notifikasjon-produsent-api.intern.nav.no/api/graphql

Vi anbefaler bruk av service discovery som beskrevet i nais dokumentasjonen.

GraphQL over HTTP

Vi implementerer GraphQL over HTTP (kun POST, ikke GET) og JSON, basert p de offisielle anbefalingene: https://graphql.org/learn/serving-over-http/.

GraphQL returnerer alltid en "well-formed" HTTP 200 OK. Dersom det er en eller flere valideringsfeil eller noe annet ugyldig vil det returneres informasjon om dette i errors feltet i response body fra server.

Vi anbefaler at dere angir correlationId i kall dere gjr mot APIet. Dette vil lette arbeidet med feilsking og oppflging. Vi plukker verdien ut fra en av flgende headere:

  • X-Request-ID
  • X-Correlation-ID
  • call-id
  • callId
  • call_id

Med dette angitt kan dere ske i kibana etter x_correlation_id.

merkelapp - String Filtrer p merkelapp. Kan ikke brukes sammen med merkelapper.
merkelapper - [String!] Filtrer p merkelapper. Kan ikke brukes sammen med merkelapp.

Response

Returns a NyKalenderavtaleResultat!

Arguments
Name Description
virksomhetsnummer - String! Hvilken virksomhet som skal motta kalenderavtalen.
grupperingsid - String!

Grupperings-id-en knytter denne kalenderavtalen til en sak med samme grupperings-id og merkelapp. Det vises ikke til brukere. Saksnummer er en naturlig grupperings-id.

Nr dere bruker grupperings-id, s er det mulig for oss presentere en tidslinje med alle notifikasjonene og status-oppdateringer knyttet til en sak.
merkelapp - String!

Merkelapp for kalenderavtalen. Er typisk navnet p ytelse eller lignende. Den vises ikke til brukeren, men brukes i kombinasjon med grupperingsid for koble kalenderavtalen til sak.

Hva du kan oppgi som merkelapp er bestemt av produsent-registeret.
